# Candidate Discovery Reference

Use this checklist when scanning a round or replay output.

## Minimal case record

- case id
- round and product
- timestamp or window
- why the case matters
- best favorable path after entry
- first meaningful adverse move
- nearby actual trade ids if any
- classification

## Ranking heuristics

- repeated pattern beats one-off beauty
- clean edge after costs beats raw excursion
- simple explanation beats bespoke narrative
- cases that suggest one reusable module beat cases that need product-specific hacks

## Red flags

- only looks good on a screenshot
- depends on future path to justify entry
- would require hardcoding timestamps
- only exists in one window with no analogs

## Tutorial-round note

- If the task is an official tutorial upload, prefer the smallest profitable patch across all provided tutorial days over a clever one-day optimization.
